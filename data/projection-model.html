Timeline Projection Model — Complete Mechanism · July 2026

Architecture

User parameters → profileToWeights() → TimelineWeights
                                     ↓
PCD + TimelineWeights → projectTimeline() → projectedSOC (Est. Arrival)
                                     ↓
monteCarloConfidence() → P10 / P50 / P90 percentiles

Two core functions:

  • profileToWeights — deterministic; converts qualitative trial characteristics into numerical month weights
  • projectTimeline — adds those months to the Primary Completion Date (PCD) to get calendar dates

The Three Weight Components

WeightMeaningDefault
submissionTime from PCD to FDA submission (data lock, NDA/BLA prep)2 months
reviewFDA review clock (filing → decision)8 (standard) / 4 (accelerated)
nccnLagTime from FDA approval → NCCN guideline inclusion5 months (fixed)
Formula: Est. Arrival = PCD + submission + review + nccnLag

Pathways (The Foundation)

Two regulatory pathways set the base review time. All adjustments are additive on top of these bases.

PathwayBase ReviewRationale
Standard 8 months Full FDA review cycle. Real-world median for standard NME reviews is ~10 months — rounded to 8 as a conservative working estimate.
Accelerated 4 months Assumes Priority Review (PDUFA ~6 months) or Accelerated Approval based on surrogate endpoint. Cuts review clock roughly in half. Enabled by Breakthrough Therapy Designation (BTD), Accelerated Approval (AA), or Priority Review (PR).

Endpoint Adjustments

Each pathway has a native baseline endpoint — the most common primary endpoint for that pathway type. Adjustments deviate from that baseline.

EndpointStandardAcceleratedWhy
PFS 0 (baseline) — Reference case for standard pathway (phase 3 RCT). No adjustment needed.
ORR −2 0 (baseline) Standard: ORR reads out faster than PFS — no need to wait for progression events, enabling earlier submission. Accelerated: ORR is the default assumption for accelerated approval (single-arm trials), so already priced in.
OS +5 +3 OS takes longest to mature — requires enough death events for significance. Standard path penalty is larger (+5) because OS trials here are typically large phase 3 studies. Accelerated path gets a smaller penalty (+3) because accelerated OS trials are often smaller or single-arm with historical comparison.

Why ORR on Standard = −2 but ORR on Accelerated = 0?
The accelerated pathway's baseline review of 4 months already assumes an ORR-based surrogate endpoint (the most common basis for accelerated approval). So an ORR trial on the accelerated path is the reference case — no further discount. On the standard path, the reference is PFS, and ORR is faster, so it gets a discount.

Enrollment Adjustments

PCD is the estimated date from ClinicalTrials.gov. If enrollment underperforms, real PCD slips. These adjustments account for that risk.

RateAdjustmentWhy
Fast0 (baseline)Reference case — rapid enrollment with no delay to PCD
Average+1Slightly slower than ideal — adds ~1 month lag in reaching target enrollment
Slow+4Significantly behind schedule — 4+ extra months just to hit enrollment targets, delaying everything downstream

Enrollment adjustments go into review (not submission) because enrollment delays push the entire timeline, including when the FDA sees the application.

Design Adjustments

Trial design affects both submission complexity (how fast you compile and file the NDA after PCD) and review complexity (how fast the FDA evaluates it).

DesignStandardAcceleratedWhy
RCT 0 (baseline) +2 (submission) Standard: RCT is the norm — no adjustment. Accelerated: RCT on accelerated path is unusual — larger, more complex data package requires more submission prep time.
SingleArm −1 (submission) 0 (baseline) Standard: Simpler dataset — no comparator arm, fewer sites, less data cleaning → −1 month submission prep. Accelerated: Single-arm is the default assumption for accelerated path — no adjustment.
Adaptive −1 (review) −1 (review) Adaptive designs (seamless phase 2/3, sample size re-estimation) shorten development and may reduce regulatory complexity — −1 regardless of pathway.

Why submission vs review? Submission adjustments affect NDA/BLA preparation speed after PCD. Review adjustments affect FDA evaluation speed after filing.

FDA Designations (BTD / AA / PR)

These don't directly modify months in profileToWeights — they determine which pathway the user selects.

DesignationEffect on Point EstimateEffect on Confidence
BTD (Breakthrough Therapy)Enables Accelerated pathway selectionTightens review distribution (FDA resources prioritized)
AA (Accelerated Approval)Enables Accelerated pathway selectionTightens submission lower bound
PR (Priority Review)Enables Accelerated pathway selectionNarrows review lower bound (faster review triggered)

Once the user selects Accelerated pathway, these designations primarily affect the Monte Carlo confidence interval (narrower distribution = higher confidence), not the point estimate.

Default Profiles

When the app has no user-saved profile for a given NCT ID, it infers from available phase data using these defaults:

PathwayEndpointEnrollmentDesignBTDAAPR
Standard (default) PFSFastRCTFalseFalseFalse
Accelerated (default) ORRFastSingleArmTrueTrueTrue

These defaults are chosen to match the most common trial archetypes for each pathway. The isDefaultStd / isDefaultAcc check in profileToWeights skips all adjustments when a profile exactly matches the default — avoiding redundant calculations.

The Core Function

export function profileToWeights(profile: TrialProfile): TimelineWeights {
  const isAcc = profile.pathway === "Accelerated";
  let review = isAcc ? 4 : 8;
  let submission = 2;

  const isDefaultStd = !isAcc && profile.endpoint === "PFS" && 
                        profile.enrollment === "Fast" && profile.design === "RCT";
  const isDefaultAcc = isAcc && profile.endpoint === "ORR" && 
                        profile.enrollment === "Fast" && profile.design === "SingleArm";

  if (!isDefaultStd && !isDefaultAcc) {
    if (profile.endpoint === "OS") review += isAcc ? 3 : 5;
    else if (profile.endpoint === "ORR" && !isAcc) review -= 2;
    if (profile.enrollment === "Slow") review += 4;
    else if (profile.enrollment === "Average") review += 1;
    if (!isAcc && profile.design === "SingleArm") submission -= 1;
    else if (isAcc && profile.design === "RCT") submission += 2;
    else if (profile.design === "Adaptive") review -= 1;
  }

  return { submission, review, nccnLag: 5 };
}

Why Additive?

The model uses additive adjustments because real-world timeline drivers are largely independent:

  • Enrollment speed doesn't affect endpoint maturation rate
  • FDA pathway doesn't affect NCCN lag
  • Trial design complexity is independent of enrollment pace

Each factor stacks its own month delta on the base. Multiplicative models would incorrectly create compounding effects between independent variables. The defaults are set so that a "typical" trial (PFS, Fast, RCT, Standard or ORR, Fast, SingleArm, Accelerated) produces realistic timelines with zero adjustment — non-default cases push the date earlier or later from that anchor.

Monte Carlo Simulation

monteCarloConfidence runs 10,000 iterations sampling each weight from a triangular distribution:

submission ~ Triangular(min, mode, max)
review     ~ Triangular(min, mode, max)
nccnLag    ~ Triangular(min, mode, max)

Total = submission + review + nccnLag  (sampled per iteration)

Results sorted to produce P10, P50, P90 percentiles
Confidence score = 1 - ((P90 - P10) / P50)

Distribution bounds are adjusted by:

  • enrollment risk slider → widens all bounds
  • CMC risk slider → widens review upper bound
  • urgency risk slider → widens NCCN lag upper bound
  • BTD → tightens submission mode and review max
  • Priority Review → tightens review min
  • AA → tightens submission min
